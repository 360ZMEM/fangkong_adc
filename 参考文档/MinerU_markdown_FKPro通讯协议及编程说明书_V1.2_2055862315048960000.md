# FKPro 通讯协议及编程说明书

# 1、概 述

SK2100、SK2300、SK2500、SK2600、SK3000 系列产品一般都同时支持三种协议，第一种 FkPro 协议，第二种为 Modbus_Tcp 协议，第三种为 Modbus_Rtu 协议。用户可根据自己选择的通讯方式、编程语言来选择其中一种通讯协议，建议选FkPro 协议，比较适合高速数据交互传输。第一种协议将在后面详细说明，后面两种协议为常用工业标准通讯协议，用户可以自查相关标准。

# 1.1、Modbus_Tcp 协议

Modbus_Tcp 协议支持该系列产品的 Ethernet、WIFI、GPRS 通讯方式。该系列产品只支持Modbus_Tcp协议里的3（读）、16（写）这两个功能代码。

# 1.2、Modbus_Rtu 协议

Modbus_Rtu 协议支持该系列产品的 RS485、RS232、ZigBee 通讯方式。该系列产品只支持Modbus_Rtu协议里的3（读）、16（写）这两个功能代码。

# 2、FkPro 协议

FkPro 为本公司开放式自定义协议，支持该系列产品的 Ethernet、RS485、RS232、WIFI、GPRS、ZigBee 等多种通讯方式。

FkPro协议把所有对模块寄存器的分为常规寄存器和串行数据寄存器。串行数据寄存器为读写串行数据的地址口，通过读写该地址，可以读取或者写入数据到模块内部的串行数据存储器区。其他寄存器都为常规寄存器。方控自定义协议对所有的常规寄存器操作都是 16bit数据宽度，串行寄存器操作则按照字节先后排列。协议里所有字段都按照先高字节，再低字节的顺序排列。

FkPro协议主要由以下4条指令及对应的返回指令组成。

# 3、采集仪内部寄存器说明

所有寄存器为16bit宽，占2个字节。读写寄存器方式都采用大端在前，小端在后模式。地址偏移值为 00-09、180 及之后的寄存器对 SK2100、SK2300、SK2500、SK2600、SK3000 都有效。地址偏移值为 10-179 之间的寄存器只对 SK2100、SK2300、SK2500、SK2600 有效。

以下说明中，“—”表示禁止，“N”表示常规寄存器，“F”表示串行寄存器。“U”表示该寄存器数据为无符号数据，“S”表示该寄存器数据为有符号数据。
