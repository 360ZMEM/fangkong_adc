# [OPEN] debug-time-drift-scope

## 背景

- 设备：`SK2301`
- 现象 1：当前时间轴比真实时间仍慢约 10%，长时间后会感觉积压
- 现象 2：波形图希望按模拟示波器逻辑显示，时间轴按 `20ms/格`
- 当前状态：真机可连接、采集链路已基本可用，但显示与实际时间仍不完全同步

## 假设

1. 当前 GUI 波形时间轴仍按配置采样率 `2000Hz` 直接映射，而设备真实有效样本率接近 `1800Hz`，导致显示时间比真实时间慢。
2. 读流泵虽然已节流，但请求周期仍偏快或偏慢，导致设备缓存与上位机显示时间基准不一致，累积形成 6 秒级延迟。
3. 消费线程解码虽然已按激活通道重组，但 `snapshot.sample_rate_hz` 仍使用名义采样率而非估计有效采样率，导致波形横轴标注不准。
4. 当前波形图固定显示 `3s`，不符合示波器阅读习惯；50Hz 信号在 3 秒窗口下过于密集，难以辨认局部周期。
5. 真机接收速率、队列长度和每秒还原时间点数量之间没有闭环标定，因此 UI 与设备真实时间存在系统性偏差。

## 计划

1. 对采集链路添加最小插桩，记录每包解码点数、累计样本率、队列长度、请求周期。
2. 运行真机采集并读取调试日志，确认“显示变慢”是时间轴映射问题还是仍有链路积压。
3. 如果证据指向时间基准问题，则引入有效采样率估计并用于波形时间轴。
4. 将波形图改为示波器风格时间轴，按 `20ms/格` 显示。
5. 修复后再运行真机验证，并比较修复前后表现。
